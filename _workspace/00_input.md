# #실시간 MVP 입력 정리

## 목표

`#실시간`은 사용자가 현재 위치 주변의 사진과 현장 상태를 제보하고, 다른 사용자는 궁금한 장소의 현재 상황을 질문할 수 있는 위치 기반 실시간 현장 확인 서비스다.

## 핵심 문제

- 사진이 실제 지금 찍힌 것인지 확인해야 한다.
- 제보자가 장소 근처에 있었는지 검증해야 한다.
- 제보가 몇 분 전 정보인지 명확히 보여야 한다.
- 허위 제보와 광고성 제보를 억제해야 한다.
- 사람 얼굴, 차량번호, 병원/관공서 민감정보 노출을 줄여야 한다.

## MVP 범위

- 웹/PWA 우선, Next.js + Supabase + Vercel 배포 가능 구조.
- 초기 지역은 울산, 부산, 경주 중심.
- 초기 카테고리는 관광지, 축제/행사장, 맛집/카페, 병원, 관공서, 주차장.
- MVP 제외: 결제, 현금성 포인트, DM, 팔로우, 업체 광고, 네이티브 앱, AI 자동 판독, 전국 자동 확장.

## 필수 화면

1. 홈
   - 검색창
   - 내 주변 실시간 제보
   - 지금 질문 올라온 곳
   - 지금 혼잡한 곳
   - 지금 한산한 곳
2. 지도
   - 내 위치 기준 주변 제보 표시
   - 장소별 상태 핀 표시
   - 혼잡도 색상 표시
3. 장소 상세
   - 장소명, 주소, 현재 상태 요약
   - 최근 사진 제보
   - 질문 목록
   - 제보하기, 질문하기 버튼
4. 제보 작성
   - 사진 업로드
   - 혼잡도: 여유 / 보통 / 혼잡 / 매우 혼잡
   - 줄 상태: 없음 / 짧음 / 보통 / 김
   - 주차 상태: 가능 / 부족 / 만차 / 모름
   - 체감 날씨: 좋음 / 비옴 / 바람셈 / 더움 / 추움
   - 한 줄 코멘트
   - 위치 인증
5. 질문 작성
   - 장소 선택
   - 질문 유형: 혼잡도 / 줄 / 주차 / 날씨 / 사진 요청 / 기타
   - 질문 내용 입력
   - 질문권 차감
6. 마이페이지
   - 내 질문권
   - 내가 올린 제보
   - 내가 작성한 질문
   - 신뢰도 점수

## 포인트/질문권 규칙

- 가입 시 질문권 3개
- 위치 인증 제보 +1
- 사진 포함 제보 +1
- 질문에 답변 +2
- 일반 질문 -1
- 사진 요청 질문 -2
- 신고로 허위 확정 시 -5

## 신뢰/안전 요구사항

- 정확한 사용자 위치는 공개하지 않는다.
- DB에는 사용자 좌표 원본 저장을 최소화한다.
- 제보에는 장소와의 거리 구간만 저장한다.
- 제보는 3시간 후 만료한다.
- 얼굴/차량번호/개인정보 노출 신고 기능을 넣는다.
- 병원/관공서 카테고리는 민감정보 업로드 경고를 표시한다.
- 사진 EXIF 제거 또는 서버 저장 전 재인코딩 정책을 문서화한다.
- Supabase RLS를 전제로 설계한다.

## UI 스타일

- 모바일 우선.
- Apple-style clean.
- 지도 + 카드형 UI.
- 밝고 신뢰감 있는 디자인.
- 버튼은 크고 직관적으로.
- 제보 작성은 10초 안에 완료 가능한 흐름.

## 성공 지표

- 장소 상세 조회 수
- 제보 작성률
- 질문 작성률
- 질문당 답변률
- 평균 답변 시간
- 사진 포함 제보 비율
- 신고율
- 재방문율
- 카카오톡 공유 수
