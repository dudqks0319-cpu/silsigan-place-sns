# #실시간 개인정보/위치정보/사진/신고 정책 초안

## 1. 개인정보 처리 원칙

- MVP는 서비스 제공에 필요한 최소 정보만 수집한다.
- 사용자의 정확한 현재 위치는 공개하지 않는다.
- 위치 인증에 사용한 원본 좌표는 검증 직후 폐기하고 DB에 저장하지 않는다.
- 사진 원본 메타데이터, 원본 파일명, GPS EXIF는 저장/노출하지 않는다.
- 민감정보가 포함된 콘텐츠는 신고 접수 즉시 숨김 또는 제한 노출을 우선 적용한다.

## 2. 위치정보 정책

### 수집 목적

- 사용자가 특정 장소 근처에서 제보했는지 확인한다.
- 주변 제보를 거리 구간 기준으로 정렬/표시한다.

### 저장 금지

- 사용자 원본 위도/경도
- GPS EXIF
- 이동 경로 또는 방문 이력
- 로그/분석 도구에 raw coordinate 전송

### 저장 허용

- 장소 ID
- 위치 인증 여부
- 거리 구간: 50m 이내, 50-150m, 150-300m, 범위 밖
- 인증 시각
- 3시간 만료 시각

### 사용자 노출

- “장소 근처 인증됨”, “약 150m 이내” 같은 구간 표현만 표시한다.
- 지도에는 사용자 좌표가 아니라 장소/제보 대상 위치만 표시한다.

## 3. 사진 업로드 정책

- 업로드 가능한 형식은 JPEG/PNG/WebP로 제한한다.
- SVG, HTML, 실행 가능한 파일, MIME 위장 파일은 금지한다.
- 서버 저장 전 EXIF를 제거하고 이미지를 재인코딩한다.
- GPS, 촬영 기기, 촬영 시각, 원본 파일명은 저장/노출하지 않는다.
- 얼굴, 차량번호, 신분증, 영수증, 진료/민원 문서, 결제정보가 보이는 사진은 금지한다.
- 문제 사진은 신고 접수 시 우선 숨김 처리 후 운영자가 검토한다.

## 4. 병원/관공서 민감정보 제한

병원/관공서 카테고리는 다음 업로드를 금지한다.

- 환자, 보호자, 민원인, 직원의 얼굴이 식별되는 사진
- 환자명, 접수번호, 대기번호, 주민등록번호, 전화번호, 주소
- 진료 서류, 처방전, 민원 서류, 신분증, 명찰, 차량번호
- 특정 개인의 방문 사실을 추정할 수 있는 사진 또는 코멘트

제품 UX 요구:

- 제출 전 민감정보 경고를 별도 표시한다.
- 병원/관공서 사진은 “사람/문서/번호판이 보이면 업로드 금지” 문구를 표시한다.
- 민감 카테고리 신고는 일반 신고보다 높은 우선순위로 운영한다.

## 5. 신고 정책

신고 유형:

- 개인정보 노출
- 얼굴/차량번호 노출
- 병원/관공서 민감정보
- 허위 위치/허위 사진
- 광고/스팸
- 폭력/혐오/불법 정보
- 기타

처리 원칙:

- 명백한 개인정보/민감정보 신고는 먼저 숨기고 나중에 판단한다.
- 신고자와 피신고자의 개인정보는 서로 공개하지 않는다.
- 반복 허위 제보자는 신뢰도 감점, 질문권 제한, 계정 제한을 적용할 수 있다.
- 신고 악용 사용자는 신고 기능 제한을 적용할 수 있다.

권장 SLA:

- 개인정보/민감정보: 접수 즉시 자동 숨김 또는 운영자 긴급 확인, 24시간 내 최종 판단.
- 허위/스팸: 72시간 내 검토.
- 긴급 삭제 요청: 운영자 알림 채널로 즉시 전달.

## 6. 삭제/보관 정책

- 제보는 기본 3시간 후 만료되어 공개 목록에서 제외된다.
- 작성자 삭제 요청 시 공개 노출을 즉시 중단한다.
- 운영자 삭제 시 DB 레코드 상태, 사진 파일, 썸네일, 캐시를 함께 정리한다.
- 법적 분쟁 또는 악용 조사 목적의 최소 감사 로그만 제한 기간 보관한다.
- 감사 로그에는 raw coordinate와 사진 원본 메타데이터를 저장하지 않는다.

## 7. 운영자 체크리스트

- [ ] 민감 신고 큐를 매일 확인한다.
- [ ] 병원/관공서 신고는 우선 처리한다.
- [ ] 삭제 처리 후 사진 URL이 더 이상 접근되지 않는지 확인한다.
- [ ] 반복 악용 계정의 제보/질문/신고 패턴을 검토한다.
- [ ] 운영 로그에 위치 원본 또는 비밀키가 남지 않는지 주기 점검한다.
